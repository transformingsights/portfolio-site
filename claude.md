# Project: Portfolio Website (Chanelle Abankwah)

## Overview
This project is a highly designed, interactive portfolio site for a multidisciplinary designer. The focus is on strong visual identity, motion, and storytelling through case studies.

The site must closely follow pre-designed wireframes and prioritize design accuracy over generic layouts.

---

## Core Goals
- Match wireframes as closely as possible
- Create a standout, memorable hero interaction
- Showcase multiple disciplines (product, branding, social, web)
- Use motion and scroll interactions intentionally (not excessively)
- Maintain a clean editorial aesthetic

---

## Key Interactions

### 1. Hero Section (Homepage)
- Pure black background
- Dense star field + subtle noise/grain texture
- Orbiting circular elements (animated)
- Circles rotate around central content
- Each circle contains labels:
  - "designer"
  - "graphic design"
  - "web designer"
  - "product"
  - "ux/ui"

### 2. Type Animation
- "I'm the ___ designer"
- Words cycle dynamically (typewriter or similar)

### 3. Scroll Text Mask Section (CRITICAL FEATURE)
- One static block of text:
  "a product designer that understands how people relate to identity, culture, and emotion through design."

Scroll behavior:
1. Text starts solid (filled)
2. Text becomes transparent (cut-out)
3. Background stays black
4. Images appear behind the text THROUGH the letters
5. Background images change 3 times while scrolling
   - NOT multiple text blocks
   - ONE text block, changing imagery behind it

## Design System

### Color
- Primary: Black & White
- Accent: Minimal (TBD, but NOT gold)
- Hero: Dark cosmic aesthetic (chalky, textured, starfield)

### Typography
- Headings: Cal Sans (custom font file provided)
- Body: Afacad

### Layout
- Editorial, minimal
- Strong whitespace usage
- Asymmetrical grid for work section
- Must match wireframes exactly

## Non-Negotiables
- Do NOT deviate from wireframes
- Do NOT introduce random design styles
- Do NOT use gold accents
- Interactions must feel intentional and polished
- Typography must match provided fonts exactly

---

## Build Philosophy
- Design fidelity > speed
- Interactions should feel premium
- Avoid overengineering — clean, intentional code
